The biggest wins were :

  • Getting the API automated testing Suite up and running in all envs.
  • Splitting the UI automation suite into two equal halves to cut back on the suite execution time
  • Being able to schedule the Jenkins jobs at a specific time so that the team does not need to login via VPN and even avoid using VPN if there are any issues when trying to kickoff the autosuite.
  • Created Auto tests for projects right before they got into production. Typically in UAT phase as opposed to the previous practice of creating the tests after release
  • Adding the newly created API methods(Eg: Policy Payee) into the API automated testing suite right after we release them.
